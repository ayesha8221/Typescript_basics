Typescript basics

function return types

  • the void return statement means the function does not have a return statement. it does not return anything and will yield undefined
  • undefined is a type in typescript and will therefore not throw an error however use "void"

functions as types

  • types that describe a function regarding the parameters and the return value
  • it is created with an arrow function "=>" but no curly braces eg. let combineValues: () => ;
  • the parameters are not variable names but their types eg. let combineValues: (a: number, b: number) => number;
  • allow us to describe which type of function we specifically want to use somewhere

function types and callbacks

  • callback is a function passed in as a parameter that should do something with the result
  • void also means that it will ignore any return even if return is specified in a callback function

the unknown type

  • the unknown type allows you to store any type in there with no errors but more restrictive than :any

the never type

  • never returns - more strict than void

The Typescript compiler

Watch mode

  • tsc <'filename'> --watch or -w automatically recompiles file

Compiling the entire project

  • compiling multiple files - navigate into folder and tsc --init and will run the whole folder and then tsc and can combine with watch mode

including and excluding files

  • tsconfig.json tells typescript how to compile these files
  • in tsconfig.json can inculde an exclude array with files you do not want to execute
  • the reverse happens with include array
  • also files array for files only to compile
